68 """ 

69 A decorator for test functions or fixtures, to parametrize them based on test cases. It works similarly to 

70 `@pytest.mark.parametrize`: argnames represent a coma-separated string of arguments to inject in the decorated 

71 test function or fixture. The argument values (argvalues in `pytest.mark.parametrize`) are collected from the 

72 various case functions found according to `cases`, and injected as lazy values so that the case functions are called 

73 just before the test or fixture is executed. 

74 

75 By default (`cases=AUTO`) the list of test cases is automatically drawn from the python module file named 

76 `test_<name>_cases.py` or if not found, `cases_<name>.py`, where `test_<name>` is the current module name. 

77 

78 Finally, the `cases` argument also accepts an explicit case function, cases-containing class, module or module name; 

79 or a list of such elements. Note that both absolute and relative module names are suported. 

80 

81 Note that `@parametrize_with_cases` collection and parameter creation steps are strictly equivalent to 

82 `get_all_cases` + `get_parametrize_args`. This can be handy for debugging purposes. 

83 

84 ```python 

85 # Collect all cases 

86 cases_funs = get_all_cases(f, cases=cases, prefix=prefix, glob=glob, has_tag=has_tag, filter=filter) 

87 

88 # Transform the various functions found 

89 argvalues = get_parametrize_args(host_class_or_module, cases_funs, debug=False) 

90 ``` 

91 

92 :param argnames: same than in @pytest.mark.parametrize 

93 :param cases: a case function, a class containing cases, a module object or a module name string (relative module 

94 names accepted). Or a list of such items. You may use `THIS_MODULE` or `'.'` to include current module. 

95 `AUTO` (default) means that the module named `test_<name>_cases.py` or if not found, `case_<name>.py`, will be 

96 loaded, where `test_<name>.py` is the module file of the decorated function. When a module is listed, all of 

97 its functions matching the `prefix`, `filter` and `has_tag` are selected, including those functions nested in 

98 classes following naming pattern `*Case*`. Nested subclasses are taken into account, as long as they follow the 

99 `*Case*` naming pattern. When classes are explicitly provided in the list, they can have any name and do not 

100 need to follow this `*Case*` pattern. 

101 :param prefix: the prefix for case functions. Default is 'case_' but you might wish to use different prefixes to 

102 denote different kind of cases, for example 'data_', 'algo_', 'user_', etc. 

103 :param glob: an optional glob-like pattern for case ids, for example "*_success" or "*_failure". Note that this 

104 is applied on the case id, and therefore if it is customized through `@case(id=...)` it should be taken into 

105 account. 

106 :param has_tag: a single tag or a tuple, set, list of tags that should be matched by the ones set with the `@case` 

107 decorator on the case function(s) to be selected. 

108 :param filter: a callable receiving the case function and returning `True` or a truth value in case the function 

109 needs to be selected. 

110 :param ids: optional custom ids, similar to the one in `pytest.mark.parametrize`. Users may either provide an 

111 iterable of string ids, or a callable. If a callable is provided it will receive the case functions. Users 

112 may wish to use `get_case_id` or other functions in the API to inspect the case functions. 

113 :param idstyle: This is mostly for debug. Style of ids to be used in the "union" fixtures generated by 

114 `@parametrize` if some cases are transformed into fixtures behind the scenes. `idstyle` possible values are 

115 'compact', 'explicit' or None/'nostyle' (default), or a callable. `idstyle` has no effect if no cases are 

116 transformed into fixtures. As opposed to `ids`, a callable provided here will receive a `ParamAlternative` 

117 object indicating which generated fixture should be used. See `@parametrize` for details. 

118 :param scope: the scope of the union fixture to create if `fixture_ref`s are found in the argvalues 

119 :param debug: a boolean flag to debug what happens behind the scenes 

120 :return: 

121 """

125 

126 # Collect all cases 

127 cases_funs = get_all_cases(f, cases=cases, prefix=prefix, glob=glob, has_tag=has_tag, filter=filter) 

128 

129 # Build ids from callable if provided. 

130 _ids = ids 

131 if ids is not None: 

132 try: 

133 # if this is an iterable, don't do anything 

134 iter(ids) 

135 except TypeError: 

136 # id this is a callable however, use the callable on the case function (not fixture_ref and lazy_values)

147 debug=debug, scope=scope) 

148 

149 if needs_inject:

171 """ 

172 Creates a glob-like matcher for the name of case functions 

173 The only special character that is supported is `*` and it can not be 

174 escaped. However it can be used multiple times in an expression. 

175 

176 :param glob_str: for example `*_success` or `*_*` 

177 :return: 

178 """ 

179 # escape all special regex characters, then find the (escaped) stars and turn them into the regex star .* 

180 re_str = re.escape(glob_str).replace("\\*", ".*") 

181 # add "end" special regex char

185 case_fun_id = get_case_id(case_fun) 

186 assert case_fun_id is not None 

187 return name_matcher.match(case_fun_id) 

188

515 """ 

516 Implements the `module=AUTO` behaviour of `@parameterize_cases`: based on the decorated test function `f`, 

517 it finds its containing module name "test_<module>.py" and then tries to import the python module 

518 "test_<module>_cases.py". 

519

521 

522 :param f: the decorated test function 

523 :return: 

524 """

526 cases_module_name1 = "%s_cases" % f.__module__ 

527 try: 

528 cases_module = import_module(cases_module_name1) 

529 except ModuleNotFoundError: 

530 # Then try `case_<name>.py` 

531 parts = f.__module__.split('.') 

532 assert parts[-1][0:5] == 'test_' 

533 cases_module_name2 = "%s.cases_%s" % ('.'.join(parts[:-1]), parts[-1][5:]) 

534 try: 

535 cases_module = import_module(cases_module_name2) 

536 except ModuleNotFoundError:

539 "cases module %r nor %r. Maybe you wish to import cases from somewhere else ? In that case" 

540 "please specify `cases=...`." 

541 % (f, cases_module_name1, cases_module_name2)) 

542 

543 return cases_module
